# テスト設計書（クオカード コーディングテスト 仕様準拠）

## 1. 方針

| レイヤー | 種別 | 目的 |
|----------|------|------|
| Service | **統合テスト**（本ドキュメント） | 仕様どおりの動作を実 DB で検証 |
| Service | 単体テスト（Mockito） | ビジネスロジックの分岐を高速に検証 |
| Application | contextLoads | Spring コンテキスト起動確認 |

フロントエンドは不要のため、UI テストは行いません。  
`http://localhost:8080/` にアクセスして 404 が表示されるのは **正常** です（ルート URL に API は定義していません）。

---

## 2. 仕様とテストの対応

| 仕様 | 統合テスト |
|------|-----------|
| 著者を RDB に登録・更新 | `AuthorServiceIntegrationTest` 登録・更新 |
| 書籍を RDB に登録・更新 | `BookServiceIntegrationTest` 登録・更新 |
| 著者に紐づく本を取得 | `AuthorServiceIntegrationTest` 書籍一覧 |
| 価格 0 以上 | DTO `@Min(0)` + DB CHECK（統合では正常系のみ） |
| 著者最低 1 人 | `BookServiceIntegrationTest` 複数著者紐付け |
| 生年月日 ≤ 今日 | DTO `@PastOrPresent`（統合では過去日付で正常系） |
| 出版済み → 未出版 不可 | `BookServiceIntegrationTest` 状態遷移 NG |
| 著者が複数書籍を執筆可能 | `BookServiceIntegrationTest` 1著者多書籍 |

---

## 3. 統合テスト一覧

### AuthorServiceIntegrationTest

| ID | テスト | 期待結果 |
|----|--------|----------|
| IT-A01 | 著者登録・更新 | DB に反映されレスポンスが返る |
| IT-A02 | 存在しない著者更新 | `NotFoundException` |
| IT-A03 | 著者の書籍一覧 | 紐づく書籍のみ返る |
| IT-A04 | 存在しない著者の書籍一覧 | `NotFoundException` |

### BookServiceIntegrationTest

| ID | テスト | 期待結果 |
|----|--------|----------|
| IT-B01 | 書籍登録（複数著者） | 書籍と著者紐付けが DB に保存 |
| IT-B02 | 存在しない著者 ID で登録 | `NotFoundException` |
| IT-B03 | 未出版 → 出版済み更新 | 正常更新 |
| IT-B04 | 出版済み → 未出版更新 | `BusinessRuleViolationException` |
| IT-B05 | 書籍詳細取得（追加 API） | 著者情報を含む |
| IT-B06 | 1 著者・複数書籍 | 2 件取得 |

---

## 4. 実行方法

```powershell
docker compose up -d
.\gradlew.bat bootRun   # 初回のみ（Flyway）
.\gradlew.bat test
```

PostgreSQL（`localhost:5432`）が起動している必要があります。  
`@Transactional` により各テスト後にロールバックされ、データは残りません。

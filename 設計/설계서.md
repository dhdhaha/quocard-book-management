| 프레임워크 | Spring Boot 4.x |

## 1. 개요

### 1.1 목적

쿠오카드(クオカード) 코딩 테스트 요구사항에 따라, 도서와 저자를 관리하는 REST API를 구축한다. 프론트엔드는 대상에서 제외하며 백엔드 API만 구현한다.

### 1.2 범위

| 대상 | 내용 |
|------|------|
| 구현 | 도서·저자 **등록·수정**, **저자에 연결된 도서 조회**, 유효성 검증, 단위·통합 테스트 |
| 미구현 | 프론트엔드, 인증·인가, 도서·저자 삭제 |
| 추가 구현 (사양 외·선택) | `GET /api/books/{id}` 도서 상세 (편의용) |

> **브라우저에서 `http://localhost:8080/` 접속 시 404(Whitelabel Error Page)가 나오는 것은 정상입니다.**  
> 이 프로젝트는 Web API만 제공하며 루트 URL·프론트엔드는 없습니다.

### 1.3 설계 방침

- 레이어드 아키텍처(Controller → Service → Repository)를 채택한다
- jOOQ를 통한 타입 안전 SQL 접근을 수행한다
- Flyway로 스키마를 버전 관리한다
- 과도한 추상화(불필요한 인터페이스, 범용 기반 클래스 등)는 피한다
- Kotlin의 Null 안전성을 활용하고, `val`을 기본으로 사용한다

## 3. 도메인 모델

### 3.1 엔티티 관계

```
┌──────────────┐       ┌────────────────┐       ┌──────────────┐
│   authors    │       │  book_authors  │       │    books     │
├──────────────┤       ├────────────────┤       ├──────────────┤
│ id (PK)      │───┐   │ book_id  (PK,FK)│───┐   │ id (PK)      │
│ name         │   └──►│ author_id(PK,FK)│◄──┘   │ title        │
│ birth_date   │       └────────────────┘       │ price        │
└──────────────┘                                 │ publication_ │
                                                 │   status     │
                                                 └──────────────┘

관계: 저자 ──(N:M)── 도서
      1명의 저자는 여러 도서를 집필 가능
      1권의 도서는 여러 저자를 가질 수 있음 (최소 1명)
```

### 3.2 저자 (Author)

| 속성 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | BIGINT | PK, AUTO INCREMENT | 저자 ID |
| name | VARCHAR(255) | NOT NULL | 저자명 |
| birth_date | DATE | NOT NULL | 생년월일 (현재일 이전) |

### 3.3 도서 (Book)

| 속성 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | BIGINT | PK, AUTO INCREMENT | 도서 ID |
| title | VARCHAR(255) | NOT NULL | 제목 |
| price | INTEGER | NOT NULL, CHECK (price >= 0) | 가격 (0 이상) |
| publication_status | VARCHAR(20) | NOT NULL | 출판 상태 |

### 3.4 출판 상태 (PublicationStatus)

| 값 | 설명 |
|----|------|
| `UNPUBLISHED` | 미출판 |
| `PUBLISHED` | 출판 완료 |

**상태 전이 규칙:**

```
UNPUBLISHED ──► PUBLISHED   ✅ 허용
PUBLISHED     ──► UNPUBLISHED ❌ 금지
```

### 3.5 도서-저자 중간 테이블 (book_authors)

| 속성 | 타입 | 제약 | 설명 |
|------|------|------|------|
| book_id | BIGINT | PK, FK → books.id | 도서 ID |
| author_id | BIGINT | PK, FK → authors.id | 저자 ID |

## 4. 데이터베이스 설계

### 4.1 Flyway 마이그레이션

| 파일 | 내용 |
|------|------|
| `V1__create_authors_table.sql` | authors 테이블 생성 |
| `V2__create_books_table.sql` | books 테이블 생성 |
| `V3__create_book_authors_table.sql` | book_authors 중간 테이블 생성 |

### 4.2 DDL

#### V1__create_authors_table.sql

```sql
CREATE TABLE authors (
    id         BIGSERIAL    PRIMARY KEY,
    name       VARCHAR(255) NOT NULL,
    birth_date DATE         NOT NULL
);
```

#### V2__create_books_table.sql

```sql
CREATE TABLE books (
    id                  BIGSERIAL    PRIMARY KEY,
    title               VARCHAR(255) NOT NULL,
    price               INTEGER      NOT NULL CHECK (price >= 0),
    publication_status  VARCHAR(20)  NOT NULL
);

CREATE INDEX idx_books_publication_status ON books (publication_status);
```

#### V3__create_book_authors_table.sql

```sql
CREATE TABLE book_authors (
    book_id   BIGINT NOT NULL REFERENCES books(id),
    author_id BIGINT NOT NULL REFERENCES authors(id),
    PRIMARY KEY (book_id, author_id)
);

CREATE INDEX idx_book_authors_author_id ON book_authors (author_id);
```

### 4.3 jOOQ 코드 생성

- Flyway 마이그레이션 실행 후, Gradle jOOQ 플러그인으로 테이블 클래스를 자동 생성한다
- 생성 위치: `src/main/kotlin` 또는 `build/generated-src/jooq`
- 사용 클래스 예: `Authors`, `Books`, `BookAuthors`, `Tables`, `Keys`

## 5. API 설계 (쿠오카드 사양)

베이스 경로: `/api`

| 필수/선택 | Method | Path | 사양상 기능 |
|-----------|--------|------|-------------|
| **필수** | POST | `/api/authors` | 저자 등록 |
| **필수** | PUT | `/api/authors/{id}` | 저자 수정 |
| **필수** | GET | `/api/authors/{id}/books` | 저자에 연결된 도서 조회 |
| **필수** | POST | `/api/books` | 도서 등록 |
| **필수** | PUT | `/api/books/{id}` | 도서 수정 |
| 선택 | GET | `/api/books/{id}` | 도서 상세 (사양 외·구현됨) |

테스트 설계: [`テスト設計書.md`](テスト設計書.md)

### 5.1 저자 API

#### POST `/api/authors` — 저자 등록

**요청:**

```json
{
  "name": "山田太郎",
  "birthDate": "1980-01-15"
}
```

**응답:** `201 Created`

```json
{
  "id": 1,
  "name": "山田太郎",
  "birthDate": "1980-01-15"
}
```

**유효성 검증:**

| 필드 | 규칙 |
|------|------|
| name | 필수, 빈 문자열 불가 |
| birthDate | 필수, 현재일 이전 |

---

#### PUT `/api/authors/{id}` — 저자 수정

**요청:**

```json
{
  "name": "山田太郎",
  "birthDate": "1980-01-15"
}
```

**응답:** `200 OK` (응답 본문은 등록 시와 동일 형식)

**에러:**

| 조건 | HTTP 상태 |
|------|-----------|
| 저자가 존재하지 않음 | 404 Not Found |
| 유효성 검증 실패 | 400 Bad Request |

---

#### GET `/api/authors/{id}/books` — 저자의 도서 목록 조회

**응답:** `200 OK`

```json
[
  {
    "id": 1,
    "title": "Kotlin入門",
    "price": 2800,
    "publicationStatus": "PUBLISHED",
    "authors": [
      { "id": 1, "name": "山田太郎", "birthDate": "1980-01-15" }
    ]
  }
]
```

**에러:**

| 조건 | HTTP 상태 |
|------|-----------|
| 저자가 존재하지 않음 | 404 Not Found |

---

### 5.2 도서 API

#### POST `/api/books` — 도서 등록

**요청:**

```json
{
  "title": "Kotlin入門",
  "price": 2800,
  "publicationStatus": "UNPUBLISHED",
  "authorIds": [1, 2]
}
```

**응답:** `201 Created`

```json
{
  "id": 1,
  "title": "Kotlin入門",
  "price": 2800,
  "publicationStatus": "UNPUBLISHED",
  "authors": [
    { "id": 1, "name": "山田太郎", "birthDate": "1980-01-15" },
    { "id": 2, "name": "鈴木花子", "birthDate": "1985-06-20" }
  ]
}
```

**유효성 검증:**

| 필드 | 규칙 |
|------|------|
| title | 필수, 빈 문자열 불가 |
| price | 필수, 0 이상 |
| publicationStatus | 필수, 유효한 enum 값 |
| authorIds | 필수, 1개 이상, 모두 존재하는 저자 ID |

---

#### PUT `/api/books/{id}` — 도서 수정

**요청:**

```json
{
  "title": "Kotlin入門 改訂版",
  "price": 3000,
  "publicationStatus": "PUBLISHED",
  "authorIds": [1]
}
```

**응답:** `200 OK` (응답 본문은 등록 시와 동일 형식)

**비즈니스 규칙:**

| 조건 | HTTP 상태 |
|------|-----------|
| 출판 완료 → 미출판으로 변경 | 400 Bad Request |
| 도서가 존재하지 않음 | 404 Not Found |
| 존재하지 않는 저자 ID 지정 | 404 Not Found |
| 유효성 검증 실패 | 400 Bad Request |

---

#### GET `/api/books/{id}` — 도서 상세 조회

**응답:** `200 OK` (등록 시와 동일 형식의 단일 객체)

**에러:**

| 조건 | HTTP 상태 |
|------|-----------|
| 도서가 존재하지 않음 | 404 Not Found |

---

### 5.3 에러 응답 공통 형식

```json
{
  "status": 400,
  "message": "出版済みの書籍は未出版に変更できません",
  "timestamp": "2026-06-20T12:00:00"
}
```

| HTTP 상태 | 용도 |
|-----------|------|
| 400 Bad Request | 유효성 검증 오류, 비즈니스 규칙 위반 |
| 404 Not Found | 리소스가 존재하지 않음 |
| 500 Internal Server Error | 예기치 않은 서버 오류 |

## 7. 비즈니스 규칙 목록

| # | 규칙 | 검증 위치 |
|---|------|-----------|
| BR-01 | 도서 가격은 0 이상이어야 함 | DTO `@Min(0)` + DB CHECK |
| BR-02 | 도서는 최소 1명의 저자를 가져야 함 | Service |
| BR-03 | 지정한 저자 ID는 모두 존재해야 함 | Service |
| BR-04 | 저자 생년월일은 현재일 이전이어야 함 | DTO `@PastOrPresent` + Service |
| BR-05 | 출판 완료된 도서를 미출판으로 변경할 수 없음 | Service |
| BR-06 | 존재하지 않는 리소스 접근 시 404 반환 | Service / Repository |

## 8. 트랜잭션 설계

| 작업 | 트랜잭션 | 이유 |
|------|----------|------|
| 저자 등록·수정 | `@Transactional` | 단일 테이블 작업 |
| 도서 등록 | `@Transactional` | books + book_authors 다중 INSERT |
| 도서 수정 | `@Transactional` | books UPDATE + book_authors 전체 삭제·재 INSERT |

도서 수정 시 저자 연결은 기존 `book_authors` 레코드를 삭제한 뒤 새로 INSERT하는 방식(Replace 방식)을 채택한다.

## 9. 테스트 설계

### 9.1 테스트 방침

| 우선순위 | 대상 | 방법 |
|----------|------|------|
| 높음 | Service 계층 비즈니스 로직 | JUnit 5 + Mockito (Repository 모킹) |
| 중간 | 유효성 검증 | `@WebMvcTest` + MockMvc |
| 낮음 | Repository 계층 | `@SpringBootTest` + Testcontainers (시간이 되면) |

### 9.2 테스트 케이스 목록

#### AuthorService

| # | 테스트 케이스 | 기대 결과 |
|---|--------------|-----------|
| T-01 | 정상적인 저자를 등록한다 | 저자가 저장되고 응답이 반환된다 |
| T-02 | 미래 생년월일로 등록한다 | 예외가 발생한다 |
| T-03 | 존재하지 않는 저자를 수정한다 | NotFoundException |
| T-04 | 저자의 도서 목록을 조회한다 | 연결된 도서 목록이 반환된다 |
| T-05 | 존재하지 않는 저자의 도서 목록을 조회한다 | NotFoundException |

#### BookService

| # | 테스트 케이스 | 기대 결과 |
|---|--------------|-----------|
| T-06 | 정상적인 도서를 등록한다 | 도서와 저자 연결이 저장된다 |
| T-07 | 저자 없이 도서를 등록한다 | 예외가 발생한다 |
| T-08 | 존재하지 않는 저자 ID로 도서를 등록한다 | NotFoundException |
| T-09 | 음수 가격으로 도서를 등록한다 | 유효성 검증 오류 |
| T-10 | 미출판 → 출판 완료로 수정한다 | 정상적으로 수정된다 |
| T-11 | 출판 완료 → 미출판으로 수정한다 | BusinessRuleViolationException |
| T-12 | 존재하지 않는 도서를 수정한다 | NotFoundException |
| T-13 | 도서 상세를 조회한다 | 저자 정보를 포함한 응답이 반환된다 |

아래는 이번 범위에서 제외하되, 향후 확장 후보로 기록한다.

| 항목 | 설명 |
|------|------|
| 삭제 API | 요구사항에 포함되지 않아 미구현 |
| 페이지네이션 | 도서 목록 조회가 요구사항에 없어 미구현 |
| 인증·인가 | 요구사항에 포함되지 않아 미구현 |
| 감사 로그 (created_at 등) | 요구사항에 포함되지 않아 미구현 |

## 1. 概要

### 1.1 目的

クオカード コーディングテストの要件に基づき、書籍と著者を管理する REST API を構築する。フロントエンドは対象外とし、バックエンド API のみを実装する。

### 1.2 スコープ

| 対象 | 内容 |
|------|------|
| 実装する | 書籍・著者の **登録・更新**、**著者に紐づく書籍の取得**、バリデーション、単体・統合テスト |
| 実装しない | フロントエンド、認証・認可、書籍・著者の削除 |
| 追加実装（仕様外・任意） | `GET /api/books/{id}` 書籍詳細（利便性のため） |

> **ブラウザで `http://localhost:8080/` を開くと 404（Whitelabel Error Page）が表示されます。正常です。**  
> 本プロジェクトは Web API のみで、ルート URL やフロントエンドはありません。

### 1.3 設計方針

- レイヤードアーキテクチャ（Controller → Service → Repository）を採用する
- jOOQ による型安全な SQL アクセスを行う
- Flyway でスキーマをバージョン管理する
- 過度な抽象化（不要なインターフェース、汎用基底クラス等）は避ける
- Kotlin の Null 安全性を活用し、`val` を基本とする

## 3. ドメインモデル

### 3.1 エンティティ関係

```
┌──────────────┐       ┌────────────────┐       ┌──────────────┐
│   authors    │       │  book_authors  │       │    books     │
├──────────────┤       ├────────────────┤       ├──────────────┤
│ id (PK)      │───┐   │ book_id  (PK,FK)│───┐   │ id (PK)      │
│ name         │   └──►│ author_id(PK,FK)│◄──┘   │ title        │
│ birth_date   │       └────────────────┘       │ price        │
└──────────────┘                                 │ publication_ │
                                                 │   status     │
                                                 └──────────────┘

関係: 著者 ──(N:M)── 書籍
      1人の著者は複数の書籍を執筆可能
      1冊の書籍は複数の著者を持つことが可能（最低1人）
```

### 3.2 著者 (Author)

| 属性 | 型 | 制約 | 説明 |
|------|-----|------|------|
| id | BIGINT | PK, AUTO INCREMENT | 著者ID |
| name | VARCHAR(255) | NOT NULL | 著者名 |
| birth_date | DATE | NOT NULL | 生年月日（現在日以前） |

### 3.3 書籍 (Book)

| 属性 | 型 | 制約 | 説明 |
|------|-----|------|------|
| id | BIGINT | PK, AUTO INCREMENT | 書籍ID |
| title | VARCHAR(255) | NOT NULL | タイトル |
| price | INTEGER | NOT NULL, CHECK (price >= 0) | 価格（0以上） |
| publication_status | VARCHAR(20) | NOT NULL | 出版状況 |

### 3.4 出版状況 (PublicationStatus)

| 値 | 説明 |
|----|------|
| `UNPUBLISHED` | 未出版 |
| `PUBLISHED` | 出版済み |

**状態遷移ルール:**

```
UNPUBLISHED ──► PUBLISHED   ✅ 許可
PUBLISHED     ──► UNPUBLISHED ❌ 禁止
```

### 3.5 書籍-著者 中間テーブル (book_authors)

| 属性 | 型 | 制約 | 説明 |
|------|-----|------|------|
| book_id | BIGINT | PK, FK → books.id | 書籍ID |
| author_id | BIGINT | PK, FK → authors.id | 著者ID |

## 4. データベース設計

### 4.1 Flyway マイグレーション

| ファイル | 内容 |
|----------|------|
| `V1__create_authors_table.sql` | authors テーブル作成 |
| `V2__create_books_table.sql` | books テーブル作成 |
| `V3__create_book_authors_table.sql` | book_authors 中間テーブル作成 |

### 4.2 DDL

#### V1__create_authors_table.sql

```sql
CREATE TABLE authors (
    id         BIGSERIAL    PRIMARY KEY,
    name       VARCHAR(255) NOT NULL,
    birth_date DATE         NOT NULL
);
```

#### V2__create_books_table.sql

```sql
CREATE TABLE books (
    id                  BIGSERIAL    PRIMARY KEY,
    title               VARCHAR(255) NOT NULL,
    price               INTEGER      NOT NULL CHECK (price >= 0),
    publication_status  VARCHAR(20)  NOT NULL
);

CREATE INDEX idx_books_publication_status ON books (publication_status);
```

#### V3__create_book_authors_table.sql

```sql
CREATE TABLE book_authors (
    book_id   BIGINT NOT NULL REFERENCES books(id),
    author_id BIGINT NOT NULL REFERENCES authors(id),
    PRIMARY KEY (book_id, author_id)
);

CREATE INDEX idx_book_authors_author_id ON book_authors (author_id);
```

### 4.3 jOOQ コード生成

- Flyway マイグレーション実行後、Gradle jOOQ プラグインでテーブルクラスを自動生成する
- 生成先: `src/main/kotlin` または `build/generated-src/jooq`
- 利用クラス例: `Authors`, `Books`, `BookAuthors`, `Tables`, `Keys`

## 5. API 設計（クオカード仕様）

ベースパス: `/api`

| 必須/任意 | Method | Path | 仕様上の機能 |
|-----------|--------|------|--------------|
| **必須** | POST | `/api/authors` | 著者登録 |
| **必須** | PUT | `/api/authors/{id}` | 著者更新 |
| **必須** | GET | `/api/authors/{id}/books` | 著者に紐づく本の取得 |
| **必須** | POST | `/api/books` | 書籍登録 |
| **必須** | PUT | `/api/books/{id}` | 書籍更新 |
| 任意 | GET | `/api/books/{id}` | 書籍詳細（仕様外・実装済み） |

テスト設計: [`テスト設計書.md`](テスト設計書.md)

### 5.1 著者 API

#### POST `/api/authors` — 著者登録

**リクエスト:**

```json
{
  "name": "山田太郎",
  "birthDate": "1980-01-15"
}
```

**レスポンス:** `201 Created`

```json
{
  "id": 1,
  "name": "山田太郎",
  "birthDate": "1980-01-15"
}
```

**バリデーション:**

| フィールド | ルール |
|------------|--------|
| name | 必須、空文字不可 |
| birthDate | 必須、現在日以前 |

---

#### PUT `/api/authors/{id}` — 著者更新

**リクエスト:**

```json
{
  "name": "山田太郎",
  "birthDate": "1980-01-15"
}
```

**レスポンス:** `200 OK`（レスポンスボディは登録時と同形式）

**エラー:**

| 条件 | HTTPステータス |
|------|----------------|
| 著者が存在しない | 404 Not Found |
| バリデーションエラー | 400 Bad Request |

---

#### GET `/api/authors/{id}/books` — 著者の書籍一覧取得

**レスポンス:** `200 OK`

```json
[
  {
    "id": 1,
    "title": "Kotlin入門",
    "price": 2800,
    "publicationStatus": "PUBLISHED",
    "authors": [
      { "id": 1, "name": "山田太郎", "birthDate": "1980-01-15" }
    ]
  }
]
```

**エラー:**

| 条件 | HTTPステータス |
|------|----------------|
| 著者が存在しない | 404 Not Found |

---

### 5.2 書籍 API

#### POST `/api/books` — 書籍登録

**リクエスト:**

```json
{
  "title": "Kotlin入門",
  "price": 2800,
  "publicationStatus": "UNPUBLISHED",
  "authorIds": [1, 2]
}
```

**レスポンス:** `201 Created`

```json
{
  "id": 1,
  "title": "Kotlin入門",
  "price": 2800,
  "publicationStatus": "UNPUBLISHED",
  "authors": [
    { "id": 1, "name": "山田太郎", "birthDate": "1980-01-15" },
    { "id": 2, "name": "鈴木花子", "birthDate": "1985-06-20" }
  ]
}
```

**バリデーション:**

| フィールド | ルール |
|------------|--------|
| title | 必須、空文字不可 |
| price | 必須、0以上 |
| publicationStatus | 必須、有効な enum 値 |
| authorIds | 必須、1件以上、全て存在する著者ID |

---

#### PUT `/api/books/{id}` — 書籍更新

**リクエスト:**

```json
{
  "title": "Kotlin入門 改訂版",
  "price": 3000,
  "publicationStatus": "PUBLISHED",
  "authorIds": [1]
}
```

**レスポンス:** `200 OK`（レスポンスボディは登録時と同形式）

**ビジネスルール:**

| 条件 | HTTPステータス |
|------|----------------|
| 出版済み → 未出版への変更 | 400 Bad Request |
| 書籍が存在しない | 404 Not Found |
| 存在しない著者IDを指定 | 404 Not Found |
| バリデーションエラー | 400 Bad Request |

---

#### GET `/api/books/{id}` — 書籍詳細取得

**レスポンス:** `200 OK`（登録時と同形式の単一オブジェクト）

**エラー:**

| 条件 | HTTPステータス |
|------|----------------|
| 書籍が存在しない | 404 Not Found |

---

### 5.3 エラーレスポンス共通形式

```json
{
  "status": 400,
  "message": "出版済みの書籍は未出版に変更できません",
  "timestamp": "2026-06-20T12:00:00"
}
```

| HTTPステータス | 用途 |
|----------------|------|
| 400 Bad Request | バリデーションエラー、ビジネスルール違反 |
| 404 Not Found | リソースが存在しない |
| 500 Internal Server Error | 予期しないサーバーエラー |

## 7. ビジネスルール一覧

| # | ルール | 検証場所 |
|---|--------|----------|
| BR-01 | 書籍の価格は 0 以上であること | DTO `@Min(0)` + DB CHECK |
| BR-02 | 書籍は最低 1 人の著者を持つこと | Service |
| BR-03 | 指定する著者 ID は全て存在すること | Service |
| BR-04 | 著者の生年月日は現在日以前であること | DTO `@PastOrPresent` + Service |
| BR-05 | 出版済みの書籍を未出版に変更できないこと | Service |
| BR-06 | 存在しないリソースへのアクセスは 404 を返すこと | Service / Repository |

---

## 8. トランザクション設計

| 操作 | トランザクション | 理由 |
|------|------------------|------|
| 著者登録・更新 | `@Transactional` | 単一テーブル操作 |
| 書籍登録 | `@Transactional` | books + book_authors の複数 INSERT |
| 書籍更新 | `@Transactional` | books UPDATE + book_authors 全削除・再 INSERT |

書籍更新時の著者紐付けは、既存の `book_authors` レコードを削除してから新規 INSERT する方式（Replace 方式）を採用する。

## 9. テスト設計

### 9.1 テスト方針

| 優先度 | 対象 | 手法 |
|--------|------|------|
| 高 | Service 層のビジネスロジック | JUnit 5 + Mockito（Repository をモック） |
| 中 | バリデーション | `@WebMvcTest` + MockMvc |
| 低 | Repository 層 | `@SpringBootTest` + Testcontainers（時間があれば） |

### 9.2 テストケース一覧

#### AuthorService

| # | テストケース | 期待結果 |
|---|-------------|----------|
| T-01 | 正常な著者を登録する | 著者が保存され、レスポンスが返る |
| T-02 | 未来の生年月日で登録する | 例外が発生する |
| T-03 | 存在しない著者を更新する | NotFoundException |
| T-04 | 著者の書籍一覧を取得する | 紐づく書籍リストが返る |
| T-05 | 存在しない著者の書籍一覧を取得する | NotFoundException |

#### BookService

| # | テストケース | 期待結果 |
|---|-------------|----------|
| T-06 | 正常な書籍を登録する | 書籍と著者紐付けが保存される |
| T-07 | 著者なしで書籍を登録する | 例外が発生する |
| T-08 | 存在しない著者IDで書籍を登録する | NotFoundException |
| T-09 | 負の価格で書籍を登録する | バリデーションエラー |
| T-10 | 未出版 → 出版済みに更新する | 正常に更新される |
| T-11 | 出版済み → 未出版に更新する | BusinessRuleViolationException |
| T-12 | 存在しない書籍を更新する | NotFoundException |
| T-13 | 書籍詳細を取得する | 著者情報を含むレスポンスが返る |
